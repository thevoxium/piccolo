A minimalistic, header-heavy deep learning framework for C++17. Features manual memory management, define-by-run computation graphs, and explicit CPU/GPU synchronization.

01

Lazy Evaluation

Operations build a graph without executing. Computation is only triggered via realize(), allowing for potential graph optimizations.

02

Explicit Sync

No magic data movement. You control exactly when data moves across the PCI-e bus with sync_to_host() and sync_to_device().

03

Hybrid Runtime

Mix and match devices. Keep weights on GPU and data preprocessing on CPU. Move subgraphs dynamically at runtime.

QUICK START

Tensory is designed to be dropped into existing C++ projects. It relies on standard BLAS for CPU operations and CUDA for GPU acceleration.

PREREQUISITES

  • C++17 Compliant Compiler
  • OpenBLAS / Accelerate (macOS)
  • NVIDIA CUDA Toolkit (Optional)

RUN EXAMPLES

# 1. Run CPU Example
make run-cpu

# 2. Run CUDA Example
make run-cuda

MINIMAL EXAMPLE

#include "src/engine.hpp"

int main() {
    int shape[] = {10, 1};
    Tensor* a = tensor_random(2, shape, DEVICE_CPU);
    Tensor* b = tensor_ones(2, shape, DEVICE_CPU);
    
    Tensor* c = tensor_add(a, b);
    
    realize(c); // Execute
    std::cout << *c << std::endl;
    
    tensor_free(c); // Free graph
}

Documentation

Tensory is a define-by-run deep learning framework. Unlike PyTorch, it is written in raw C++ with manual memory management, giving you absolute control over allocation and lifetime.

Tensor Management

Included from src/tensor.hpp

Tensor* tensor_create(int ndim, int* shape, Device device)
Allocates a new tensor struct. Memory is allocated immediately on the specified device.

Params:
- device: DEVICE_CPU or DEVICE_GPU
void tensor_free(Tensor* t)
Frees the memory (data, grad, shape, strides) associated with the tensor.
Note: This does not free parents in the graph. Use free_graph for that.
Tensor* tensor_from_data(int ndim, int* shape, float* data, Device d)
Creates a tensor and copies the provided float array into it.

Engine & Autograd

Included from src/engine.hpp

void realize(Tensor* t)
Triggers the Forward Pass. Topologically sorts the computation graph ending at t and executes the kernels for any node that hasn't been realized yet.
void backward(Tensor* root)
Triggers the Backward Pass.
  1. Calls realize(root).
  2. Sets root gradient to 1.0.
  3. Propagates gradients via chain rule to all ancestors.
void zero_grad(Tensor* t)
Sets the gradient buffer of the tensor to zero.

Device Control

Functions to manage memory movement between Host (CPU) and Device (GPU).

void set_graph_device(Tensor* root, Device device)
Recursively traverses the graph and moves all tensors to the specified device. Handles allocation and data copying automatically.
void sync_to_host(Tensor* t)
Checks if the tensor data is dirty on the GPU. If so, copies it back to the CPU t->data pointer. Essential before printing or saving.

Binary Operations

Standard element-wise or matrix operations.

tensor_add(a, b), tensor_sub(a, b)
Element-wise addition/subtraction. Shapes must match.
tensor_mm(a, b)
Matrix Multiplication. Input A must be (M, K) and B must be (K, N). Returns (M, N).

Unary Operations

Element-wise math functions.

tensor_sin(t) tensor_cos(t) tensor_exp(t) tensor_log(t) tensor_sqrt(t) tensor_pow(t, exp) tensor_scale(t, k) tensor_neg(t)
